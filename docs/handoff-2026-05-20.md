### Phase C: Cross-box restore validation

Tested the runbook by restoring on `ge103-5` (a different H800 box):

- pigz extract: ~3-5 min
- 10/10 imports PASS
- `torch.cuda.nccl.version() = (2, 27, 5)` loaded from venv via DT_RPATH ✓
- NOT loaded from system NCCL 2.27.3 (which is at `/usr/lib/...` on `ge103-5`)

Important learning about the NCCL alignment check:

> The old `ctypes.CDLL('libnccl.so.2')` check goes through ld.so cache and
> finds the system NCCL, which can be a different version than what torch
> uses without anything actually being broken. Use `torch.cuda.nccl.version()`
> + `/proc/self/maps` instead — that's the ground truth for what torch loads.

Doc `docs/h800-venv-backup.md` Step 4 updated to use the correct check.
